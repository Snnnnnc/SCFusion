# 训练问题修复总结

## 问题诊断结果

### 1. ✅ 归一化问题（已修复）

**问题描述：**
- 每个样本的每个patch的每个通道独立归一化
- 训练集和验证集/测试集使用了不同的归一化标准
- 导致数据分布不一致，影响模型性能

**修复方案：**
- 在 `experiment.py` 中添加了 `compute_normalization_stats()` 方法
- 只使用训练集计算全局的 mean 和 std
- 验证集和测试集使用相同的统计量进行归一化
- 修改了 `dataset.py` 中的归一化逻辑，支持使用全局统计量

**修改文件：**
- `experiment.py`: 添加归一化统计量计算和传递
- `dataset.py`: 修改归一化逻辑，支持全局统计量

### 2. ✅ Loss Function 和 Weights（无需修复）

**检查结果：**
- Loss function (`base/loss_function.py`) 正确支持 `sample_weights`
- Trainer (`trainer.py`) 在计算 loss 时正确传递了 `sample_weights`
- 如果 batch 中有 'weight' 字段，会被用于加权 loss

**结论：**
- 当前实现是正确的
- 如果 weights 是伪标签的置信度，使用它们是合理的

### 3. ⚠️ 类别不平衡问题（需要处理）

**问题描述：**
- 类别不平衡比例：10.13:1
- 类别分布：
  - 类别 0: 48.1% (4124个样本)
  - 类别 1: 25.1% (2158个样本)
  - 类别 2: 16.9% (1447个样本) ⚠️
  - 类别 3: 4.7% (407个样本)
  - 类别 4: 5.2% (445个样本)

**类别2被错判为类别1的原因：**
- 类别2的样本数量相对较少（16.9%）
- 模型倾向于预测样本数量多的类别
- 类别2和类别1的特征可能相似
- 类别2的正确分类率只有 17.8%，61.9% 被错判为类别1

**建议的解决方案：**
1. **使用类别权重（推荐）**
   ```bash
   # 在 main.py 中使用 class_weights 参数
   python main.py -class_weights "1.0,2.0,3.0,5.0,5.0" ...
   ```
   权重可以根据类别频率的倒数来计算

2. **使用 Focal Loss**
   - 修改 loss function 为 'focal'
   - Focal Loss 可以自动处理类别不平衡

3. **数据增强或重采样**
   - 对少数类别进行数据增强
   - 使用 SMOTE 等过采样技术

4. **分析特征差异**
   - 分析类别2和类别1的特征差异
   - 可能需要额外的特征或更复杂的模型

## 类别权重计算示例

```python
# 类别分布
class_counts = [4124, 2158, 1447, 407, 445]
total = sum(class_counts)
num_classes = len(class_counts)

# 计算权重（平衡所有类别）
class_weights = [total / (num_classes * count) for count in class_counts]
# 结果: [0.52, 1.00, 1.49, 5.30, 4.84]

# 归一化到合理范围
max_weight = max(class_weights)
class_weights_normalized = [w / max_weight for w in class_weights]
# 结果: [0.10, 0.19, 0.28, 1.00, 0.91]
```
